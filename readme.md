# C Programming — Embedded Systems

> *"C gives you enough rope to hang yourself. In embedded systems, that rope is the only thing holding everything together."*

---

## Why C for Embedded Systems?

**Direct Hardware Access** — C lets you manipulate registers and memory addresses directly, no abstraction layers.

**Predictable Memory** — No garbage collector, no hidden allocations. You control every byte.

**Deterministic Execution** — Compiles to lean, predictable machine code. Critical for ISRs and real-time timing.

**Near-Zero Overhead** — No runtime, no VM. The binary on the MCU is almost exactly what you wrote.

**Universal Toolchain Support** — Every MCU vendor ships a C compiler. XC8, GCC, IAR — C runs everywhere.
